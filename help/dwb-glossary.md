# Glosario del área de trabajo de datos{#data-workbench-glossary}

**Alerta**

En el área de trabajo de datos, un mensaje o informe que se activa automáticamente cuando una métrica alcanza un umbral definido dentro de un intervalo de tiempo especificado. Por ejemplo: se puede crear una alerta que emita un mensaje de correo electrónico si el número de vistas de página durante un período de 24 horas cae por encima o por debajo de una cantidad especificada.

**Analista**

Persona que realiza análisis, define informes o utiliza de otro modo el área de trabajo de datos.

**Arquitecto**

Persona que determina cómo se capturarán, procesarán y organizarán los datos para análisis e informes. Esta persona suele tener una experiencia considerable en la configuración de la plataforma de Adobe® para uso de los analistas.

**Gráfico de barras**

Forma de gráfico que utiliza barras rectangulares de distintos tamaños para mostrar comparaciones entre dos o más elementos.

**Prueba comparativa**

Medición o norma que sirve de punto de referencia para medir o juzgar a otros. En el área de trabajo de datos, una referencia ilustra la comparación entre el valor asociado con una selección (un subconjunto seleccionado) y el valor que se mostraría si no se realizara la selección.

**Cardinalidad**

Número de elementos de un conjunto. El número de elementos de una dimensión de datos de Adobe se denomina cardinalidad de la dimensión.

**Punto de comprobación**

Hora a la que se escribió una copia de un conjunto de datos de Adobe para fines de backup o recuperación. El término también se refiere al conjunto de datos que se escribe durante una operación de punto de comprobación.

**Chi-square**

Una prueba estadística para determinar la probabilidad de que una desviación observada del evento o resultado esperado se produzca por casualidad. En el área de trabajo de datos, el cuadrado más cercano de Chi es al 100%, menor es la probabilidad de que la desviación se produzca únicamente por casualidad.

**Clickstream**

Término informal que hace referencia a la secuencia de páginas que un usuario solicita durante la exploración de un sitio Web. La información del flujo de navegación puede ayudar a los propietarios del sitio a comprender cómo utilizan el sitio los visitantes y qué páginas solicitan con mayor frecuencia. Los propietarios del sitio pueden capturar los flujos de clics de los visitantes a sus sitios, pero no pueden capturar los flujos de clics fuera del sitio (excepto mediante cookies de terceros o etiquetas de página) porque dichos pedidos son registrados por servidores Web externos.

**Leyenda de color**

Leyenda del área de trabajo de datos que muestra la rampa de color de una métrica seleccionada. Las leyendas de color habilitan la codificación de colores de las visualizaciones según varias métricas. Las visualizaciones con codificación de color facilitan la detección de anomalías, excepciones y tendencias.

**Rampa de color**

En una leyenda de color, el rango de colores utilizado para representar el rango de valores posibles para una métrica. Cuando se aplica una leyenda de color a una visualización, los elementos gráficos de la visualización (por ejemplo, las barras de un gráfico de barras) se codifican con colores según los valores representados por la rampa de color.

**Clave común**

Variable común que forma una relación entre las filas de dos conjuntos de datos tabulares. Por ejemplo: un ID de producto puede actuar como clave común entre una tabla de variables de cadena de consulta y una tabla de datos de producto de un sistema de administración de inventario.

**Intervalo de confianza**

Intervalo de valores que tiene una probabilidad específica de contener la tasa o la tendencia. Los intervalos de confianza del 80% (valor p = 0,20), 95% (valor p = 0,05) y 99% (valor p = 0,01) son los intervalos de confianza más utilizados. (Fuente: `http://www.nci.nih.gov/statistics/glossary`)

**Nivel de confianza**

La probabilidad de que el error de muestra en un resultado de un estudio se encuentre dentro de un rango especificado, generalmente expresado en términos de errores estándar (por ejemplo, 1 error estándar equivale a 68% de probabilidad, 2 errores estándar equivale a 95,4% de probabilidad). (Fuente: `http://www.magazine.org/research/3410.cfm`)

**Mapa de conversión**

En el área de trabajo de datos, un tipo de visualización en la que los elementos se trazan en el eje x según el valor de la métrica Conversión.

**Tasa de conversión**

En el área de trabajo de datos, el porcentaje de sesiones durante las que se produjo un evento de valor. La tasa de conversión se calcula dividiendo el número de sesiones en las que se produjo un evento de valor por el número total de sesiones.

**Correlación**

Una medida numérica de la fuerza de una relación lineal entre dos variables aleatorias. Las variables que tienden a moverse hacia arriba o hacia abajo juntas están positivamente correlacionadas, mientras que las variables que tienden a moverse en direcciones opuestas están correlacionadas negativamente. Consulte también Coeficiente de correlación.

**Coeficiente de correlación**

Valor numérico que significa la fuerza de una relación lineal entre dos variables aleatorias. Consulte también Correlación.

**Dimensión contable**

Una dimensión en la que se puede contar discretamente el número de elementos que contiene la dimensión. Las dimensiones contables pueden tener dimensiones secundarias de los siguientes tipos: Contable, Numérico, Simple, De varios a muchos y Denormal.

**Transformación de filas cruzadas**

En el servidor del área de trabajo de datos, una transformación de datos que permite incorporar en un cálculo los datos de varios registros de eventos para un visitante (realizados en diferentes momentos).

**Crosstab**

En el área de trabajo de datos, una visualización tabular que muestra las métricas asociadas con las intersecciones de dos dimensiones.

**Cubo**

Una estructura de datos multidimensional o un grupo de celdas de datos organizadas según las dimensiones de los datos. Por ejemplo, una hoja de cálculo ejemplifica una matriz bidimensional con datos organizados en filas y columnas. En una hoja de cálculo, cada fila o columna es una dimensión. Una matriz tridimensional se puede visualizar como un cubo con cada dimensión formando un lado del cubo, incluyendo cualquier fracción paralela a ese lado. Las matrices dimensionales superiores no tienen metáforas físicas, pero pueden utilizarse para organizar los datos de manera que los usuarios piensen en su empresa. También se conoce como matriz hipercubible, multidimensional o multidimensional.

**Tablero**

Espacio de trabajo que se crea para su visualización en lugar de interacción. Los tableros proporcionan el estado &quot;de un vistazo&quot; mediante la visualización de indicadores de rendimiento clave que son adecuados para un gerente o operador particular que supervisa uno o varios objetivos comerciales. Consulte también Espacio de trabajo.

**Extracción de datos**

La aplicación no guiada o interactiva de una recopilación de procedimientos matemáticos a los datos históricos de las empresas en un esfuerzo por encontrar perspectivas en forma de correlaciones y otras relaciones estadísticas.

**Subconjunto de datos**

Característica que permite a los usuarios seleccionar fácilmente un subconjunto de un conjunto de datos para utilizarlo en análisis en línea o sin conexión. (Un subconjunto de datos es una parte del conjunto de datos basada en un filtro).

**Data Warehouse**

Base de datos diseñada para apoyar la toma de decisiones en las organizaciones. Un almacén de datos generalmente contiene grandes cantidades de datos no volátiles, con variantes de tiempo y orientados a temas concretos, estructurados para consultas rápidas en línea y resúmenes administrativos.

**Unidad de procesamiento de datos**

Tipo de servidor del área de trabajo de datos que procesa, almacena y sirve datos de un conjunto de datos de Adobe. Un DPU puede, opcionalmente, almacenar los archivos de registro VSL que contienen los datos de origen desde los que se construye el conjunto de datos o puede recibir esos datos desde una unidad de servidor de archivos (FSU) del servidor de área de trabajo de datos. Un DPU es el tipo de servidor del área de trabajo de datos con el que interactúan directamente el área de trabajo de datos y los clientes de Report®.

**Conjunto de datos**

Datos cargados y procesados por el servidor del área de trabajo de datos. El conjunto de datos representa los datos que se pueden transmitir al área de trabajo de datos o al informe para análisis, informes y alertas. Físicamente, el conjunto de datos reside en el archivo temp.db. Cada equipo servidor del área de trabajo de datos (o clúster de servidores del área de trabajo de datos) mantiene un conjunto de datos.

**Datos de conjunto de datos**

Los datos creados y almacenados en un conjunto de datos del servidor del área de trabajo de datos. Incluye los datos del evento y los datos de integración que se admiten o crean en el conjunto de datos. También incluye cualquier información derivada de esos datos, según lo determinado por los archivos de configuración que definen ese conjunto de datos. Los datos del conjunto de datos se pueden volver a crear reprocesando los datos del evento y de la integración utilizando los mismos archivos de configuración o diferentes. (Los archivos de configuración son archivos del sistema que se administran como parte de un perfil de Adobe).

**Registros de conjunto de datos**

Estos registros de datos de eventos admitidos en un conjunto de datos de Adobe después de realizar el filtrado y otro procesamiento. También se denominan entradas de registro procesadas.

**Esquema de conjunto de datos**

Una visualización en el área de trabajo de datos que muestra el esquema del conjunto de datos que respalda el perfil seleccionado actualmente.

**Espacio de almacenamiento de datos**

Cantidad de datos (en bytes) introducidos o creados en un conjunto de datos de Adobe. El conjunto de datos almacena estos datos en un archivo llamado Temp.db en un equipo servidor del área de trabajo de datos. Los datos de Temp.db son transitorios y se pueden volver a crear reprocesando los datos de origen (es decir, los datos del evento y los datos de integración) con los archivos de configuración correspondientes.

**Decodificador**

Componente del servidor del área de trabajo de datos que lee datos de eventos de varias fuentes y genera datos que se utilizan para producir el conjunto de datos. La salida de un decodificador puede utilizarse como entrada para cualquiera de las funciones de &quot;procesamiento de registros&quot; del servidor del área de trabajo de datos. Los tipos de decodificador incluyen el decodificador Sensor (para cargar datos de varias versiones de Sensor), el decodificador Expresión regular (para cargar datos de archivos planos delimitados) y el decodificador ODBC (para cargar datos de orígenes de datos ODBC).

**Dimensión**

Conjunto de elementos, todos los cuales son de un tipo similar en la percepción del usuario. Los elementos definen un conjunto de categorías en las que se pueden agrupar los datos. Por ejemplo, los elementos lunes, martes, miércoles, jueves, viernes, sábado y domingo forman una dimensión &quot;Día de la semana&quot;.

**Elemento Dimensión**

Categoría individual dentro de una dimensión. Por ejemplo, una dimensión &quot;Día de la semana&quot; contendría los elementos individuales lunes, martes, miércoles, jueves, viernes, sábado y domingo.

**Leyenda de dimensión**

En el área de trabajo de datos, la leyenda que enumera las dimensiones que se han definido en el conjunto de datos (o que se han derivado de él). Cuando se realiza una selección en una visualización, la leyenda Dimensión identifica las dimensiones cuyos valores difieren del valor de referencia en una cantidad estadísticamente significativa.

**Explorar en profundidad**

Una técnica analítica específica mediante la cual el usuario navega entre los niveles de datos que van desde los más resumidos (hasta los más detallados) hasta los más detallados (descendentes). Por ejemplo, al ver los datos de ventas para Norteamérica, una operación de exploración en profundidad en la dimensión Región podría mostrar Canadá, Estados Unidos y México. Un nuevo análisis en profundidad en Canadá podría mostrar Toronto, Vancouver, Montreal, etc.

**Datos del evento**

Los datos recopilados por los sensores u otros medios (por ejemplo, un archivo de registro de servidor web), que constituyen la entrada principal al servidor del área de trabajo de datos. Cada registro de datos de eventos representa un registro de transacciones o una sola instancia de un evento.

**Dimensión ampliada**

Dimensión basada en datos extendidos. Los datos ampliados son datos que superan lo mínimo necesario para formar un registro de datos de evento válido. Los datos ampliados se pueden agregar a un registro de datos de eventos cuando se captura el evento original o se pueden incorporar desde otras fuentes y agregarse al registro de datos del evento como datos de integración. Cualquier dimensión basada en estos datos adicionales se considera &quot;ampliada&quot;.

**Unidad de servidor de archivos (FSU)**

Un tipo de servidor del área de trabajo de datos cuya función es únicamente recibir datos de eventos de uno o varios servidores de sensores o repetidores y proporcionar datos a una o más unidades de procesamiento de datos (DPU) del servidor del área de trabajo de datos para su uso en la construcción de conjuntos de datos de Adobe. Las FSU optimizan la transferencia de datos de eventos a las DPU y son considerablemente más rápidas que los file servers normales. El uso de una FSU reduce los costos de hardware al permitir que los datos de registro se almacenen en hardware de almacenamiento de menor costo y reduce la complejidad administrativa al permitir que varios sensores apunten a un único servidor del área de trabajo de datos.

**Evaluación de consulta incremental**

Proceso patentado mediante el cual el servidor del área de trabajo de datos proporciona resultados de consulta inmediatos a un usuario según una muestra aleatoria proyectada de toda la población. Con este proceso, el servidor refina de manera incremental la precisión de la consulta considerando más datos hasta que se hayan tenido en cuenta todos los datos y se haya obtenido un recuento exacto.

**Datos de integración**

Los datos de integración son datos externos de bases de datos corporativas o archivos de búsqueda que se pueden combinar con datos de eventos para crear el conjunto de datos. En general, los datos de integración se utilizan para aumentar los datos de eventos adquiridos por Sensor. (Conceptualmente, se puede pensar en usar datos de integración para rellenar registros de datos de eventos con columnas de información adicionales).

**Leyenda**

Ventana del área de trabajo de datos que proporciona detalles explicativos sobre las visualizaciones que se muestran en el área de trabajo. Los tipos de leyendas incluyen leyendas de color, leyendas de dimensión y leyendas de métricas. Al igual que cualquier ventana del área de trabajo de datos, las ventanas Leyenda pueden generarse y distribuirse mediante el informe.

**Gráfico de líneas**

Tipo de visualización en el área de trabajo de datos que representa las métricas de una dimensión específica como puntos sucesivos en el eje x de un gráfico y, a continuación, conecta los puntos con las líneas. Un gráfico de líneas es una forma particularmente eficaz de visualizar una métrica en una dimensión basada en el tiempo.

**Condición de entrada de registro**

Condición que determina si se incluirá un registro de datos de evento (una entrada de registro) en el conjunto de datos. Por ejemplo: una condición de entrada de registro podría especificar que sólo se admitirán en el conjunto de datos los registros de datos de eventos asociados con un sitio Web en particular. Las condiciones de entrada de registro se especifican en el archivo de configuración de procesamiento de registros del servidor del área de trabajo de datos.

**Dimensión de varios a varios**

En la plataforma de Adobe, una dimensión que tiene una relación &quot;varios a varios&quot; con una dimensión contable principal. Una dimensión &quot;varios a varios&quot; representa un conjunto de valores para cada elemento de su dimensión principal. Por ejemplo, en el sitio, la dimensión Frase de búsqueda tiene una relación de varios a varios con su elemento principal, la dimensión Sesión (es decir, una Sesión puede tener cualquier número de frases de búsqueda y una Frase de búsqueda puede tener cualquier número de Sesiones).

**Enmascaramiento**

Característica del área de trabajo de datos que permite a los analistas ocultar temporalmente elementos que no desean incluir en un análisis.

**Media**

Promedio aritmético de un conjunto de números. La suma de los datos dividida por el tamaño de la muestra.

**Mediana**

Un número que separa la mitad más alta de una muestra, una población o una distribución de probabilidad de la mitad más baja. La mitad de la población tendrá valores inferiores o iguales a la mediana y la mitad de la población tendrá valores iguales o buenos a la mediana.

**Métrica**

En Adobe, una fórmula con nombre que describe cómo calcular un valor cuantitativo a partir de los datos del conjunto de datos. En el sitio, por ejemplo, la métrica &quot;Sesiones por visitante&quot; representa una fórmula que divide el recuento de Sesiones por el número de visitantes.

**Leyenda de métrica**

Ventana del área de trabajo de datos que muestra las métricas definidas por el perfil activo. Una leyenda de métrica muestra el valor de cada métrica calculada a partir del conjunto de datos o de la selección actual (si una selección está activa en el espacio de trabajo, la leyenda Métrica muestra los valores del subconjunto seleccionado en lugar de todo el conjunto de datos). Al igual que cualquier ventana del área de trabajo de datos, las leyendas de las métricas se pueden generar y distribuir mediante el informe.

**Hoja de cálculo de métricas**

Ventana del área de trabajo de datos que permite a los analistas definir sus propias métricas para un conjunto de datos. Una hoja de cálculo de métrica es similar a una hoja de cálculo. Mediante la sintaxis de fórmula del área de trabajo de datos, los analistas pueden introducir expresiones que describan los valores cuantitativos que desean derivar del conjunto de datos. Por ejemplo, un analista podría definir una métrica que muestre el porcentaje de visitantes que vieron una página desde un dominio en particular. Al igual que las métricas normales, las fórmulas de una hoja de cálculo Métrica funcionan en el subconjunto seleccionado cuando una selección está activa en el espacio de trabajo. Al igual que cualquier ventana del área de trabajo de datos, las hojas de cálculo de métricas se pueden generar y distribuir por informe.

**Nueva condición de visitante**

Condición que determina si se crea un nuevo ID de seguimiento cuando se presenta un registro de datos de evento en el servidor del área de trabajo de datos.

**Nodo**

Agrupación de uno o varios elementos discretos en una única entidad lógica. En un archivo de configuración (.cfg), un nodo es un elemento que contiene parámetros relacionados. Consulte también Parámetro y Vector. En el área de trabajo de datos, un nodo de un mapa de proceso representa una sola página o un grupo definido de páginas.

**Dimensión numérica**

En la plataforma de Adobe, una dimensión que tiene valores numéricos ordenados y una relación uno a varios con una dimensión contable principal. Generalmente, una dimensión numérica representa una propiedad numérica de los elementos de la dimensión principal. Las dimensiones numéricas se utilizan frecuentemente para definir métricas “sum”.

**Relación uno a varios**

Relación entre dos dimensiones de datos en la que un solo elemento de una dimensión está (o puede estar) relacionado con uno o más elementos de la otra dimensión.

**Ordinal**

Ser o indicar un orden numérico en una serie. (Fuente: `http://wordnet.princeton.edu/perl/webwn?s=ordinal`) Si una dimensión se ordena de forma ordinaria en el área de trabajo de datos, los elementos de la dimensión se muestran en el orden en que se representan internamente.

**Outliers**

En un conjunto de datos, un valor tan alejado de otros valores de la distribución que su presencia no se puede atribuir a la combinación aleatoria de causas casuales.

**Superposición de página**

Tipo de visualización en el área de trabajo de datos que codifica con colores los vínculos de una imagen de una página Web según una métrica especificada. Puede utilizar una visualización de Página excesiva para identificar rápidamente qué vínculos de una página atraen la atención de los visitantes (y cuáles la llevan a otras páginas del sitio) y cuáles no. También puede utilizarla para comprender el &quot;valor&quot; (medido por eventos de valor) que los distintos vínculos de una página generan para su sitio.

**Condición de vista de página**

Una opción en una transformación del servidor del área de trabajo de datos que permite incluir o excluir registros de eventos como vistas de página en función de su tipo de contenido o contenido real. Puede utilizar esta opción, por ejemplo, para excluir los registros de eventos pertenecientes a solicitudes HTTP incorrectas (por ejemplo, aquellas que produjeron un código de estado 404) o solicitudes que devuelven determinados tipos de contenido (por ejemplo, solicitudes de imagen). Si la opción Condición de vista de página no filtra un registro de evento HTTP, dicho registro representará una vista de página en el conjunto de datos.

**Visualización del explorador de rutas**

Tipo de visualización en el área de trabajo de datos que permite a un analista explorar interactivamente una secuencia de eventos (como vistas de página) en una sesión de visitante o en varias sesiones de visitante.

**Servidor principal**

En un clúster de servidores del área de trabajo de datos, el servidor del área de trabajo de datos que intercambia las comunicaciones entre clientes (como el área de trabajo de datos y el informe) y los demás servidores del clúster. El servidor principal también funciona como centro de coordinación administrativo del clúster. Mediante las capacidades de sincronización de perfiles del servidor del área de trabajo de datos, los cambios que un administrador realiza en el servidor primario se propagan automáticamente a los demás servidores del clúster. Un servidor primario es una DPU del servidor del área de trabajo de datos.

**Mapa de procesos**

Tipo de visualización en el área de trabajo de datos que permite a un analista comprender el flujo de tráfico entre páginas o nodos en un sitio Web. Un mapa de procesos muestra información sobre páginas específicas (como el número de sesiones durante las cuales se vieron las páginas) y también ilustra el volumen de tráfico entre páginas o nodos.

**Servidor de procesamiento**

En un clúster de servidores del área de trabajo de datos, los servidores del área de trabajo de datos que están controlados por la información de configuración del servidor primario. Un servidor de procesamiento procesa los datos de eventos en un conjunto de datos y responde a consultas de clientes como, por ejemplo, el área de trabajo de datos y el informe. Al responder a una consulta, un servidor de procesamiento divide (&quot;particiones&quot;) la responsabilidad de completar la consulta entre los servidores del clúster. Cuando los demás servidores completan tres partes de la consulta, el servidor de procesamiento combina (&quot;desparticiones&quot;) sus resultados y devuelve el resultado combinado al cliente.

**Perfil**

Conjunto de archivos de configuración que contiene las reglas para crear un conjunto de datos con un propósito de análisis específico. Un perfil también define artículos como métricas, dimensiones derivadas, espacios de trabajo, informes, visualizaciones y leyendas que permiten a los analistas interactuar con el conjunto de datos y obtener información de él. Un perfil se puede estructurar generalmente para un sitio Web (como en un perfil para www.mysite.com) o se puede adaptar para un tipo de usuario concreto (como el perfil de usuario &quot;Marketing&quot; o el perfil de usuario &quot;Finanzas&quot;).

**Administrador de perfiles**

Herramienta administrativa interactiva en el área de trabajo de datos que permite a un administrador u otro usuario administrar los archivos de configuración asociados a un perfil.

**Expresión regular**

Fórmula que describe o coincide un conjunto de cadenas según determinadas reglas de sintaxis. Las expresiones regulares (a menudo abreviadas como regexp, regex o regxp) se utilizan para buscar y manipular cuerpos de texto basados en ciertos patrones. La notación de expresión regular se originó en los primeros editores de Unix y obtuvo un uso generalizado en otras utilidades de Unix como vi y Perl. En la actualidad, las expresiones regulares son compatibles con muchos editores de texto, lenguajes de secuencias de comandos y otras herramientas de manipulación de texto. El servidor del área de trabajo de datos incluye un motor de expresión regular.

**Mapa de retención**

Tipo de visualización en el área de trabajo de datos que representa los elementos en el eje x según el valor de la métrica Retención.

**Diagrama de puntos**

Tipo de visualización en el área de trabajo de datos que representa los datos bivariables como puntos en un gráfico. Un diagrama de puntos tiene dos fragmentos de datos para cada elemento que se grafica. Por ejemplo: un diagrama de puntos de las métricas Tasa de conversión y Visitantes para un conjunto de 10 páginas resultaría en 10 puntos distintos en el diagrama.

**Selección**

Característica del área de trabajo de datos que permite a un analista restringir el conjunto de datos recuperados y mostrados para informes o análisis. Se puede realizar una selección interactivamente dentro del área de trabajo de datos haciendo clic en los elementos mostrados en una o varias dimensiones (al hacer clic con el botón izquierdo se selecciona un elemento, al hacer clic con el botón derecho se anula la selección de un elemento). Las selecciones también se pueden realizar definiendo filtros que seleccionan determinados elementos en dimensiones específicas.

**Dimensión simple**

En la plataforma de Adobe, una dimensión que tiene una relación uno a varios con una dimensión contable principal. Por ejemplo: Referente de visitante es una dimensión simple cuyo elemento principal es la dimensión contable, Visitante. Un visitante solo tiene un referente de visitante. Sin embargo, un referente de visitante puede tener varios visitantes (es decir, un referente de visitante puede estar relacionado con muchos visitantes).

**Suavizado**

Inferencia matemática de una curva en varios puntos de un gráfico de líneas, que se utiliza para ilustrar una línea de tendencia más significativa en puntos de datos relativamente dispersos.

**Fuente**

En el servidor del área de trabajo de datos, un recurso que contiene datos de eventos que se pueden utilizar para crear un conjunto de datos. Los descodificadores de Adobe buscan fuentes para que los datos de eventos se descodifiquen y se utilicen en el servidor del área de trabajo de datos.

**Datos de origen**

En el servidor del área de trabajo de datos, los datos que son la entrada de uno de sus descodificadores. Los datos de origen se pueden introducir en el decodificador de sensores (que carga los datos de origen de varias versiones de Sensor), el decodificador de expresiones regulares (que carga los datos de origen de archivos planos delimitados) y el descodificador ODBC (que carga los datos de origen de los orígenes de datos ODBC).

**Gráfico de tabla**

Tipo de visualización en el área de trabajo de datos que muestra los datos en formato de tabla. Los valores de las métricas en un gráfico de tabla se pueden expresar numéricamente o representar en formato de barra.

**Serie temporal**

Gráfico que muestra cómo una propiedad o un valor determinados cambian con el tiempo.

**ID de seguimiento**

Identificador que distingue de forma exclusiva las entidades principales que se analizan en un conjunto de datos de Adobe. Se puede crear una ID de seguimiento a partir de varias fuentes, incluido un ID único de una cookie de cliente web, un número IP y un hash de agente de usuario o un nombre x.509. Aunque estas son algunas fuentes comunes para una ID de seguimiento, se puede utilizar cualquier valor que sea capaz de identificar de forma exclusiva las entidades que rellenan la dimensión Visitante en el conjunto de datos.

**Transformación**

Método para cambiar el valor de una variable mediante alguna operación matemática. En el servidor del área de trabajo de datos, por ejemplo, los analistas pueden utilizar la transformación dividida para dividir los pares nombre-valor de una cadena de consulta en variables individuales.

**Gráfico de barras bidimensionales**

Tipo de visualización en el área de trabajo de datos que muestra simultáneamente dos dimensiones y hasta dos métricas en una vista gráfica tridimensional.

**Leyenda de valor**

Ventana del área de trabajo de datos que permite a un analista asociar un valor monetario con un evento seleccionado y ver los resultados cuando se suma ese valor monetario para todas las instancias del evento seleccionado en el conjunto de datos.

**Visitante**

Dimensión del conjunto de datos que identifica la entidad que generó el evento. Cada miembro de la dimensión Visitante se identifica mediante un ID de seguimiento único. En el sitio, por ejemplo, la ID de seguimiento se deriva generalmente de una ID única dentro de la cookie del cliente. En Llamada, el ID de seguimiento puede ser el número de teléfono del llamante.

**Referente del visitante**

El primer referente HTTP para un visitante dentro del lapso de tiempo de un conjunto de datos de Adobe.

**División de visitantes**

Característica en el servidor del área de trabajo de datos que permite dividir a los visitantes con grandes cantidades de datos de eventos entre dos ID de seguimiento. La división de visitantes se utiliza para evitar que los datos de eventos se filtren desde el conjunto de datos cuando un visitante supera el número máximo configurado de eventos por visitante (un parámetro que se establece para garantizar el rendimiento correcto del sistema). Aunque la división artificial del visitante aumenta artificialmente el número de visitantes en el conjunto de datos, no aumenta el número total de registros de eventos, lo que garantiza que el número total de eventos contables (por ejemplo, vistas de página, reservaciones) se mantenga preciso.

**VSL**

Archivo de registro. Tipo de archivo en el que el servidor del área de trabajo de datos almacena los datos de eventos que reciben del sensor. Los archivos VSL se comprimen, pero se pueden generar en un formulario sin comprimir mediante la funcionalidad de transformación de datos. VSL es la extensión de archivo para un archivo de registro.

**Análisis de hipótesis**

Tipo de análisis utilizado para comprender las consecuencias de cambiar ciertas variables en un modelo de datos observando el efecto que los cambios en dichas variables tienen en otros datos del modelo.

**Workspace**

En el área de trabajo de datos, un área de trabajo es un contenedor para una tarea de análisis y visualización concreta. Un espacio de trabajo puede contener varias visualizaciones), todas las cuales funcionan con un conjunto común de datos (es decir, todas las visualizaciones del espacio de trabajo representan el mismo conjunto de resultados de consulta). Cuando un analista realiza una operación de subconjunto o filtrado en una visualización, el subconjunto seleccionado se refleja en todo el espacio de trabajo.

**Escritorio**

Área &quot;principal&quot; dentro de la interfaz de usuario del área de trabajo de datos que le permite organizar y acceder a todos los espacios de trabajo e informes. También permite crear y guardar espacios de trabajo e informes nuevos y actualizados en el servidor del área de trabajo de datos para que otros usuarios que utilicen el mismo perfil también puedan acceder a ellos.
